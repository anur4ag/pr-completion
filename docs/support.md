# Support

## How to get help

Primary support channel: **GitHub Issues** on the project repository.

Planned issue tracker (after the public repository exists):

[https://github.com/anur4ag/pr-completion/issues](https://github.com/anur4ag/pr-completion/issues)

Use issues for:

- install or upgrade problems on Claude Code or Codex;
- incorrect skill behavior relative to documented authority;
- documentation gaps;
- feature requests.

Before filing:

1. Confirm prerequisites (Python 3.10+, Git, authenticated `gh`, supported harness version).
2. Note plugin version (`VERSION` / installed plugin metadata) and OS.
3. Include the skill name, truncated logs, and whether the PR is private (do not paste secrets).

## What to expect

This is an open-source project. Marketplace listing identity is **Traycer**; the GitHub repository is owned by [anur4ag](https://github.com/anur4ag).
Responses are best-effort.
There is no paid SLA and no dedicated support mailbox operated as a product service.

## Security issues

**Do not** file public issues for vulnerabilities.

Follow [SECURITY.md](../SECURITY.md). The only private channel is **GitHub private vulnerability reporting** on the public repository, enabled and verified at publication time (ticket 5). Until then that channel is planned, not live, and there is no profile-email fallback.
