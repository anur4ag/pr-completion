# Support

Use [GitHub Issues](https://github.com/anur4ag/pr-completion/issues) for install failures, behavior that differs from the documented skill authority, documentation gaps, and feature requests.

## Before filing

1. Confirm Python 3.10+, Git, authenticated `gh`, and a supported harness version.
2. Record the plugin version, operating system, and invoked skill.
3. Include the smallest useful redacted log and the terminal watcher state.
4. State whether the PR is private, but do not paste repository secrets, tokens, private keys, or sensitive source.

## Security reports

Do not file public issues for vulnerabilities. Follow the repository [security policy](../SECURITY.md) and use [GitHub private vulnerability reporting](https://github.com/anur4ag/pr-completion/security/advisories/new).

## Support terms

This is an open-source project. Support is best-effort, with no paid SLA or dedicated product-support mailbox. The marketplace publisher identity is **Traycer**; the GitHub repository is owned by [anur4ag](https://github.com/anur4ag).
